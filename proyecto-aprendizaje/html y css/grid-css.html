Aprendiendo Grid CSS

Introduccion

CSS Grid es otra manera de maquetar tu documentacion web, lo cual puede lograr muchos diseños diferentes y no biene a remplazar la otras maquetaciones si no a complementarlos.

Lo que destaca en Grid css es que puedes dividir paginas en areas o regiones principales (sistema bidimencional) para definir la relacion en tamaños, posicion y capas entre partes conmo un tablero de ajedrez.

Conceptos Basicos CSS

Aqui en Grid CSS necesitamos entender los conceptos basicos.

Al igual que flexbox vamos a tener:

  • contenedor Padre(Grid Container)

    Es a la propiedad que le aplicaremos display: grid;.

  • Elementos Hijos(Grid Items)

    Van a ser las celdas (Las celdas se forman al haber una intercepcion entre una columna y una fila).

  • Lineas de Cuadricula(Grid Lines)

    Son las lineas que forman las celdas eso quiere decir que siempre debes añadir uno mas para que formen celdas completas.

  • Pista de Cuadricula(Grid Track)

    Determina si sera toda un fila o toda una columna.

  • Celda de Cuadricula(Grid Cell)

    La celda es cada intercepcion entre una fila y columna.

  • Area de Cuadricula(Grid Area)

    Van a ser areas cuadriculares, quiere decir que puede ser rectangulo o cuadrado perfecto, combinar celdas.

Grid Explicita

Existes diferentes manera para definir una Grid, la mas fundamental es la Grid explicita

Porque se llama Grid explicita?

Porque explicitamente vamos a definir numero de filas y numero de columnas que tendra nuestra Grid.

grid-template-colums

Nos permite definir el numero de columnas

grid-template-row

Nos permite definir el numero de filas

fraccion

Es un valor para Grid y significa una fraccion el espacio sobrante, eso quiere decir que toda la parte que no este definida sera para el que tenga la fraccion fr

gap

Es una propiedad nueva que remplaza a estas 2 propiedades obsoletas (ya no se usan) grid-row-gap, grid-column-gap y el shorthand grid-gap

Esta propiedad sirve para separar las celdas de Grid

Codigo

  
    .grid-explicit {
      display: grid;
      /*grid de 3x3*/
      grid-template-columns: 50% 100px 1fr;
      grid-template-rows: 2rem 20vh 30%;
      /*grid de 5x4*/
      grid-template-columns: repeat(5, 20%);
      grid-template-rows: repeat(4, auto);
      grid-template-columns: repeat(5, 1fr);
      grid-template-rows: repeat(4, 1fr);
      /*Grid de 4x5*/
      grid-template-columns: 20% repeat(2, 30%) 20%;
      grid-template-rows: repeat();
      /*propiedades obsoletas*/
      grid-row-gap: 1rem;
      grid-column-gap: 1rem;
      grid-gap: 1rem 1rem;
      /*nueva propiedad de las obsoletas*/
      gap: 1vh 1vw;
      gap: 0;
    }

Posicionamiento con Grid Lines

Ahora aprenderemos a como acomodar elementos ocupando numero de celdas con la ayuda de las grid lines.

Propiedades:

  • grid-row-start & grid-row-end

    Sirve para poner en que celda estara este elemento de las lineas (va desde cierto numero elegido hasta hasta que quieras que sea el final) 

      
    .gridline .item:nth-child(10 {
      grid-row-start: 2;
      grid-row-end: 3;
    }

  • grid-column-start & grid-column-end

    Sirve para poner en que celda estara este elemento en las columnas (va desde cierto numero elegido hasta hasta que quieras que sea el final) 

      
    .gridline .item:nth-child(10 {
      grid-column-end: 2;
      grid-column-start: 3;
    }

  • grid-column & grid-row

    Es una atajo para las propiedades anteriores, para poder poner los dos valores de cada propiedad debes añadir / en medio. 

      
    .gridline .item:nth-child(10 {
      grid-column: 3 / 4;
      grid-row: 2 / 3;
    }

  • grid-area

    Grid-area es una atajo de grid-column y grid-row y el orden de los valores es: primero el grid-row y luego el grid-column 

      
    .gridline .item:nth-child(10) {
      grid-area: 2 / 3 / 3 / 5;
    }

Con estas propiedades tambien podemos agrandar celdas o combinarlos, para completar espacios de sobra. 

  
    .gridline .item:nth-child(10) {
      grid-row: 2 / 3;
      grid-column: 3 / 5;
    }

Hay otra manera de combinar o agrandar celdas usando el span 

  
    .gridline .item:nth-child(12) {
      grid-column: span 2;
      grid-row: span 3;
      grid-column: 1 / span 2;
      grid-row: 1 / span 2;
    }

Recomendacion

Si vas a definir las posiciones de tus elementos en las partes de las areas, lo mejor sera que definas todas las areas ya que el flujo de la grid (su algoritmo) puede posicionarlos de una mala forma.

Posicionamiento con nombres de Grid Lines

Ya vimos a como posicionar con grid line esta vez sera lo mismo pero con la tecnica linea en nombres

Como funciona?

Para poder hacer esto debemos agregar a las propiedades de Posicionamiento [] y su respectivo nombre (que no sea largo)

Recomendacion

Lo recomendable es no usar esta tecnica de nombre ya que es mas dificil de entender y genera una gran linea de texto aparte no hay diferencias con usar los valores anteriores y no podemos usar los shorthand de posicionamiento-grid-lines

    .grid-line-names {

      grid-template-row: [linea-r1] 1fr [linea-r2] 1fr [linea-r3] 1fr [linea-r4];

    .grid-line-names .item:nth-child(3) {
      grid-row: linea-r3 / linea-r4;

Posicionamiento con Grid Areas

Una grid areas forman areas rectangulares en las celdas, esto nos puede servir cuando queremos maquetar la forma de la pagina web.

Propiedades

  • grid-template-areas

    Sirve para declarar el area con un nombre como si fuera unas celdas (recomendable hacerlos con espacios) 

      
    .grid-areas {
      display: grid;
      grid-template-columns: 1fr 200px;
      grid-template-rows: 100px repeat(2, 1fr) 60px;
      grid-template-areas:
        "header header"
        "content sidebar"
        "content sidebar"
        "footer footer";
    }

  • grid-areas

    Sirve para poder aplicar lo que hicimos en grid-template-areas con sus respectivos nombres declarados. 

    
  .header {
    grid-area: header;
  }

  .content {
    grid-area: content;
  }

  .sidebar {
    grid-area: sidebar;
  }

  .footer {
    grid-area: footer;
  }

Con las grid areas tambien podemos dejar espacios en blancos sin elementos en las maquetaciones, solo debemos poner un punto en la parte que no queremos de grid-template-areas 


  .grid-areas }
  grid-template-areas:
    "header header"
    "content sidebar"
    "content ." <---- es este punto
    "footer footer";

Luego de ver esto nos damos cuenta de porque grid no remplaza a flexbox ya que grid esta pensado para la maquetacion y flexbox para centrar elementos dentro de grid

Grid Implícita. Grids de bloque y de línea

Grid Implicita

La Grid Implicita es la sobra o el resto de elementos de una Grid explicita (reticula ya definida sus columnas y filas) y las Grid Implicitas por lo general toman un valor en automatico (se trataran de ajustar al tamaño del contenedor y si no hay mas espacio, se desbordara).

Grids de bloque y de línea

Al igual que flexbox las grid tienen su forma de linea y de bloque:

  • display: grid; <---bloque

  • display: inline-grid <---linea

Si la grid esta en linea podemos introducir varias cajas con grid a la mismo ancho(calcular el tamaño)

Codigo


  .grid-implicit {
    display: grid;
    display: inline-grid;
    grid-template-columns: repeat(4, 1fr);
    grid-template-rows: repeat(3, 200px);
  }

Flujo de la Grid (Grid Flow)

El flujo de la grid permite controlar de como se reparten los elementos dentro de la celda, por defecto se generan filas pero tambien podemos cambiar el flujo.

grid-auto-flow

Sirve para cambiar el flujo de la grid, si en columna o linea, es como si fuera flex-direction de flexbox.

grid-auto-columns & grid-auto-rows

Sirven cuando sobra espacio en la reticula y queremos cambiar el tamaño de las celdas sobrantes en menor tamaño para no desbordarse

Codigo


  .grid-flow {
    display: grid;
    /*grid de 5x3*/
    grid-template-columns: repeat(5, 1fr);
    grid-template-rows: repeat(3, 150px);
    grid-auto-flow: columns; predeterminado es rows
    grid-auto-columns: 5px; predeterminado es auto
    grid-auto-rows: 150px; predeterminado es auto
  }

Orden y Alineación de Grid Items

Orden de Grid Items

Ahora vamos a ordenar los elementos hijos dentro de la Grid, para esto vamos usar la misma propiedad que se usaba en flexbox para alinear.

order

Hace ordenar los elementos de la caja grid o flexbox, el valor predeterminado es 0 y puede aceptar valores negartivos (-1, 0, 1), recuerda que el valor menor ira primero y el mayor ira ultimo. 


  .grid-order {
    display: grid;
    grid-template-columns: repeat(3, 1fr);
    grid-template-rows: repeat(2, 1fr);
  }

  .grid-order .item:nth-child(1) {
    order: 1;
  }

  .grid-order .item:nth-child(2) {
    order: 2;
  }

  .grid-order .item:nth-child(5) {
    order: -1;
  }

Alineación de Grid Items

Ahora vamos a aprender a alinear las celdas solas tambien las filas completas (Grid Track)

justify-items

Lo que hace justify-items es justificar la alineacion de los items en columna (x) y usa acepta estos valores : stretch(default), start, end, center.

align-items

Lo que hace align-items es justificar la alineacion de los items en fila (y) y acepta estos valores: stretch(default), start, end, center.

justify-self

Pero si solo quieres alinear un item idependientemente debes usar la propiedad justify-self 


  .grid-align {
    display: grid;
    grid-template-columns: repeat(3, 200px);
    grid-template-rows: repeat(2, 200px);
    /*justify-items alinea en el eje X*/
    justify-items: center;
    /*align alinea en el eje Y*/
    align-items: center;
  }

  .grid-align .item:nth-child(4) {
    justify-self: start;
  }

Tamaños Máximos y Mínimos de Grid Tracks

Existe una funcion que se llama grid-min-max que nos permite tener maximos tamaños y minimos tamaños en cada uno de los tracks

minmax()

Esta propiedad sirve para poder hacer que la grid sea mas dinamica, acepta 2 valores (la primera es para el valor minima del elemento y el segundo es para el valor maximo)

min-content & max-content

Son 2 valores que puede aceptar el minmax() y su utilidad es para reducir o aumentar el minimo o maximo del contenido(las letras), si la palabra es larga no se cortara pero si los espacios de la oracion, estos dos valores los puedes usar en los tamaño maximo o minimo cualquiera de los 2.

Codigo


  .grid-min-max {
    grid-template-columns: repeat(4, 1fr);
    grid-template-columns: repeat(4, minmax(100px, 200px));
    grid-template-columns: repeat(4, minmax(min-content, 200px));
    grid-template-columns: repeat(4, minmax(100px, min-content));
    grid-template-columns: repeat(4, minmax(100px, max-content));
    grid-template-columns: repeat(4, minmax(max-content, 200px));
    grid-template-columns: repeat(4, minmax(MIN-content, max-content));
  }

Grids Dinámicas. Responsive sin Media Queries

Grids Dinamicas

Vamos a aprender 2 valores que nos ayudaran al asignar al tamaño de las columnas y filas, estos 2 valores nos serviran para realizar resposive Design sin la necesesidad de meter Media Queries.

auto-fill

Sirve para rellenar la caja de la grid con tracks generados dinamicamente y funcionar si es fila o columna.

auto-fit

Sirve para ajustar la grid con los tracks existentes y funcionar si es fila o columna.

Importante

  • Si usamos la funcion minmax() como segundo valor en una columna o fila y esta el auto-fill, generara espacios disponibles cuando quieras agregar otros items.

  • Si usamos la funcion minmax() como segundo valor en una columna o fila y esta el auto-fit, ajustara las items dentro del contenedor.

Codigo


  .grid-dynamics {
    display: grid;
    /*grid de 4cx?f*/
    grid-template-columns: repeat(4, 100px);
    /*auto-fill rellena la grid con tracks generados dinamicamente*/
    grid-template-columns: repeat(auto-fill, 100px);
    /*auto-fit Ajusta la grid con los tracks generados dinamicamente*/
    grid-template-columns: repeat(auto-fit, 100px);
    grid-template-columns: repeat(auto-fill, minmax(100px, 1fr));
    grid-template-columns: repeat(auto-fit, minmax(100px, 1fr));
  }

Sub Grids

El problema de que no se pueden juntar las reticulas en las Grid Anidadas se puede solucionar gracias a las sub Grids, pero esta caracteristica no esta soportada en todos los navegadores.

Pasos para que funcione:

  1. Definir explicitamente el tamaño que ocupará el elemento que aplicara subgrid dentro del contenedor padre, es decir definir las propiedades grid-column y grid-row

  2. Aplicar display grid al elemento que aplicara subgrid

  3. Aplicar el valor de subgrid a las columnas, a las filas o ambas depende de como se requiera

Codigo


  .grid {
    display: grid;
    /*grid de 3cx2f*/
    grid-template-columns: repeat(3, 1fr);
    grid-template-rows: repeat(4, 1fr);
  }
  .subgrid {
    /*Paso 1*/
    grid-columns: span 3;
    grid-rows: 1 / 3;
    /*Paso 2*/
    display: grid;
    /*Paso 3*/
    grid-template-columns: subgrid;
    grid-template-rows: subgrid;
  }

resumen general

mostraremos las propiedades mas inportantes y resumire el contenido:

Propiedades

  • display: grid;

    activar el grid

  • grid-template-columns & grid-auto-rows

    definir columnas o filas

  • gap

    separa celdas

  • posicionamiento grid line

    grid-column & grid-row acomodan elementos

  • posicionamiento grid areas

    grid-template-areas & grid-area acomodar elementos

  • grid-auto-flow & grid-auto-columns & grid-auto-rows

    sirven para el flujo de la grid

  • order

    ordenar elementos

  • justify-items & align-items & justify-self

    alinear celdas

  • justify-content & align-content

    alinear tracks

  • funciones y valores:

    • 1fr

    • dense

    • span

    • minmax()

    • min-content & max-content

    • repeat()

    • auto-fit & auto-fill
