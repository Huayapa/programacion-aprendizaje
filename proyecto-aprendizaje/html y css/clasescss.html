Sintaxis Basica

Una regla de css consta de 2 partes:

  • el selector: es cualquier etiqueta html ( clases, id, mas adelante se mostrara mas detalladamente).
  • bloque de declaraciones: es cada uno de los atributos que podemos modificar.
    -Forma de sintaxis:
    atributo: valor;
    -Forma de mas palabras:
    atributo-de-mas-de-dos-palabras: otro-valor;

ejemplo:


              h1 {                (el h1 es el selector)
               color: orangered;                 (el color y el background-color 
               background-color: skyblue;          son bloques de declaraciones)
               }

Formas de invocacion CSS

Son diferentes formas de agregar estilos, existen tres formas:

  • Estilos en linea - a travez del atributo style aplicar estilos, ejemplo y codigo:
    style="font-weight: bold; color: orange" (ponerlo en una etiqueta)
  • Estilos internamente - donde vamos a aplicar al interior del documento html (creando la etiqueta <style> en <head>), ejemplo y codigo: 
                    
                  <style>
                      h2 {
                        color: yellowgreen;
                      }
                  </style>
  • definicion de hoja externa(recomendable y la ideal forma) - es crearlo por otra carpeta o archivo y para agregar de manera externa usar esto:
    <link rel="stylesheet" href="lacarpetadelcss.css" />

La forma recomendable y de buena practica no dice que debemos usar el css en una hoja externa. Para poder vincular una hoja externa dentrp de otra hoja externa que usas, debes usar el:
@import url(nombre de la carpeta)
Pero esto es de mala practica ya que a la hora de ejecutarlo el navegador tendra que leer todo y con el @import (codigo bloqueante) este se paralizara hasta que termine de descargar todo ese archivo.

selectores de atributos

Los selectores de atributo nos van a permitir aplicar estilos a travez de un atributo y tienen diferentes variantes. Para poder aplicarlo debemos poner entre corchete[] el atributo. 


            .selectores-atributo a[href] {
                color: orangered;
              }

tipos de variantes

  • * - aplica el estilo si contiene la palabra que pusistes
    • 
              .selectores-atributo a[href*="notas"] {
                  font-size: 20px;
                }
  • ^ - aplica estilos solo si empiezan con la palabra que tu quieras 
    • 
              .selectores-atributo a[href^="http:"] {
                  text-decoration: none;
              }
  • $ - aplica el estilo solo si contiene el final igual
    • 
              .selectores-atributo a[href$=".com"] {
                  background-color: black;
              }
  • ~ - aplica el estilo si contiene el texto, cuando estan separados con espacios en blanco 
    • 
              .selectores-atributo a[class~="cursos"] {
                  background-color: midnightblue;
              }
  • | - aplica el estilo si contiene el texto, cuando estan separados con espacios en blanco y que despues del texto lleve un guion medio - 
    • 
              .selectores-atributo a[class|="menu"] {
                font-style: italic;
              }

Pseudoclases:

es un selector que marca los elementos que están en un estado específico. existen un monton de pseudoclases, mostraremos los mas usados o los que mejor te servira.
mayor informacion: Pseudoclasses

ejemplo de pseudoclases con formularios:
Aqui mostraremos algunos pseudoclases que nos serviran para los formularios

  • :required - estara en su color hasta que se complete el formulario (ojo debe tener el atributo required para esto). 
    
                .form-pseudoclases *:required {
                    border-color: yellow;
                  }
  • :valid - aplicara el color cuando se complete el formulario correctamente. 
    
                .form-pseudoclases *:valid {
                    border-color: green;
                }
  • :invalid - aplicara el color cuando pongas un error en el formulario. 
    
                .form-pseudoclases *:invalid {
                    border-color: red;
                }
  • :focus - aplicara el color cuando estas interactuando con el formulario. 
    
                .form-pseudoclases *:focus {
                    background-color: skyblue;
                }
  • :disable - aplicara el color cuando el input este desabilitado. 
    
                .form-pseudoclases *:disabled {
                    border-color: grey;
                }


Ejemplos con el nav (navegador) con enlaces

EL Algoritmo de CSS

El Algoritmo de css es la forma de como aplica los estilos el CSS al documento html, Es importante esto para no sufrir problemas de que no se apliquen los estilos correctamente o simplemente no aplique nada.

Lista de 3 Algoritmo

  • La Cascada
  • La Especificidad
  • La Herencia

La Cascada:

La cascada es un Algoritmo para aplicar estilos en CSS y tienen cierto orden que debes tomar en cuenta:

  1. El Origen del Codigo:
    • El User Agent (los estilos predeterminado del navegador)
    • Personalizaciones del usuario (ejemplo como poner tema blanco y oscuro o el zoom del navegador)
    • Los estilos del autor (osea nosotros como creador de html y css)
  2. La Especificidad del Selector - es la manera en el cual el navegador decide que valores de css son mas inportante para tu elemento
  3. Orden de aparición - es de como va leyendo las lineas de codigo(de arriba a abajo y de izquierda a derecha)

La Especificidad:

La especificad es la forma de como los navegadores deciden que elementos son mas inportantes y, por lo tanto, se aplicaran (Aqui la cascada no tiene inportancia, ya que el que tenga mayor peso es el que ganara).

La Especificidad es el peso que tiene un selector y se calcula de esta manera;

etiquetas y pseudoelementos 0,0,0,1
clases, atributos y pseudoclases 0,0,1,0
identificadores 0,1,0,0
Estilos en linea 1,0,0,0
!important Rompe la especificad

Lo recomendable en crear un estilo y que el peso no sobre pase de 0,0,2,# (para evitar problemas), recordar que por cada selector de la tabla aumentara dependiendo del peso que tenga.

La Herencia:

Es la capacidad de un selector el cual determina la propiedad de un elemento padre y se transmite a sus hijos. se parece a la herencia genetica

Modelos de Caja

Es una caracteristica fundamental en CSS, ya que condiciona el diseño a todas las paginas web,para el ejemplo es mejor crearlo en otro documento html

Ejemplo de box model

Los modelos de caja contienen las siguientes propiedades:

  1. El contenido (content) - es el contenido del elemento HTML (texto).
  2. El borde (border) - Lo que delimita cada uno de los elementos HTML.
  3. El relleno (padding) - Son las distancias intenas (la distancia del borde con el contenido).
  4. El margen (margin) - Son las distancias externas (la distancia entre el elemento a otros elementos padres o hermanos).

Dentro del modelo de caja hay que considerar sus 4 lados.

  1. Arriba (top)
  2. Derecha (right)
  3. Abajo (bottom)
  4. izquierda (left)

Border

    Caja de linea VS Caja de bloque

    caracteristicas:

    Caja de linea

    • Ocupan el espacio necesario para mostrar su contenido
    • No tienen dimensiones modificables (alto, ancho)
    • Permiten otros elementos a su lado
    • Padding y margin solo empujan a los elementos adyacentes en horizontal, nunca en vertical

    Caja de bloque

    • Ocupan todo el ancho disponible, lo que genera saltos de linea
    • Tienen dimensiones modificables (alto y ancho)
    • No permiten otros elementos a su lado(aunque lo especifiques)

    Ejemplo

    Hola soy una caja de linea.Hola soy otra caja de linea.
    Hola soy una caja de bloque.
    Hola soy otra caja de bloque.

    Codigo

            .caja-bloque {
              background-color: darkcyan;
              color: white;
              width: 500px;
              height: 100px;
              border: medium solid black;
              padding: 16px;
            }

            .caja-linea {
              margin: 20px;
            }

              margin: 40px;

    Propiedad Display

    Display es la propiedad que nos mostrara las cajas y especifica si esta es una caja de linea o bloque.

    Lista de elementos Display

    • display: none; - el elemento es completamente removido del elemento(no se puede ver pero el contenido esta cargado).

      • El display none nos puede servir a la hora de un formulario, ya que si te salen avisos de que esta mas algo, para poder ocultarlos es necesario el none.

    • display: inline; - el elemento solo usa el espacio requerido.
    • display: block; - el elemento usa caja de bloque.
    • display: inline-block; - tiene caracteristicas de linea pero se puede modificar sus dimensiones (alto y acho) como uno de bloque.

      • el display block y inline nos pueden servir para cambiar a los elementos de bloque y linea en viseversa(linea y bloque) a la hora de maquetar.

    • display: list-item; - sirve para las etiquetas <li>(tanto listas ordenadas como desordenadas)

    Posicionamiento de CSS

    entrar a posicionamiento css

    La propiedad FLOAT y POSITION nos permiten modificar la posicion de cualquier documento HTML.

    Elementos flotantes

      aqui mayor informacion
    • float
    • clear

    Tipos de Posicionamiento

    • static (esto es el predeterminado)
    • relative
    • absolute
    • fixed
    • sticky

    Para mover los elementos posicionados podemos usar estas propiedades:

    • top (vertical - eje Y)
    • bottom (vertical - eje Y)
    • left (horizontal - eje X)
    • right (horizontal - eje X)
    • z-index (profundidad - eje Z)

    Las propiedades top y left van a tener mayor preferencia que bottom y right respectivamente.

    al usar el top, left , etc su margen no aumenta los pixeles (osea del contenido al margen) si no se alejan del contendido(de forma contraria)

    Estas cinco propiedades no funcionan con el valor static (ya que es el predeterminado)

    Posicionamiento Static

    Es el posicionamiento predeterminado y significa que el elemento respeta el flujo del documento HTML, por lo que no se puede mover ni posicionar. 

    
            .static {
              background-color: yellow;
              position: static;
              no se mueven con el top y left 
              porque es estatico
              top: 16px;
              left: 16px;
            }

    Ejemplo

    Soy una caja que no me puedo mover

    Posicionamiento Relativo

    El elemento relative conserva sus dimensiones y posicion original en el flujo del documento. Si se mueve toma como referencia su posicion original.

    Se puede mover con pixeles positivos y negativos

    al usar el top, left , etc su margen no aumenta los pixeles (osea del contenido al margen) si no se alejan del contendido(de forma contraria) 

    
            .relative {
              background-color: turquose;
              position: relative;
              top: -20px;
              left: 50px;
            }

    Ejemplo

    Posicionamiento Absoluto

    El elemento pierde sus dimensiones y posicion original en el documento. Cuando lo muevas puede tomar como referencia 2 elementos:

    1. El primer ancestro (cuando este dentro de otra caja osea una div, section, article, etc...) con posicionamiento relativo (debe tener las propiedades absolute el ancestro)
    2. Si no encuentra un ancestro relativo, se mueve hacia el documento HTML 
    
            .absolute {
              background-color: lightcoral;
              position: absolute;
              top: 10px;
              left: 5px;
            }

    Posicionamiento Fixed

    El elemento pierde sus dimensiones y posicion original en el flujo del documento. Si se mueve toma como referencia solo al documento HTML y su posicion estara fijo (aunque lo muevas con el scroll, el elemento no se movera).

    Esto nos podria servir para que la barra principal este fija en todo el documento. 

    
              .fixed {
                background-color: lightgreen;
                position: fixed;
                top: 30px;
                left: 50px;
                }

    Posicionamiento Sticky

    Es una combinación de posicionamiento relative y fixed, para este posicionamiento las propiedades top, left, bottom y right no pueden mover el elemento, sirven como un punto de referencia(mientras no lleguen a ese valor, el elemento se comportara como relative, cuando llega se convierte en fixed).

    Para que este comportamiento funcione el elemento sticky debe ser hijo directo de body o su elemento contenedor debe tener dimensiones definidad y solo será sticky dentro de las dimensiones de su contenedor padre.

    Propiedad z index

    Es la propiedad que controla la profundidad del posicionamiento, su valor predeterminado es auto, puede aceptar números positivos, negativos y ceros.

    A mayor valor del elemento estara mas al frente, a menor valor del elemento estara mas al fondo.

    Un elemento padre nunca podrá estar sobre sus elementos hijos, pero si estos tienen un valor negativo si pueden estar sobre sus elementos pádres (el elemento padre no debe tener definido el z-index).

    Ejemplo

    Lo recomendable es de poner el z-index de en 5 en 5 o 10
